# article-workflow 项目功能分析报告

## 📋 项目概况

**项目名称**：article-workflow  
**版本**：v2.0 (2025-03)  
**代码规模**：~3500+ 行高质量代码  
**状态**：生产就绪，优化完成

## 🏗️ 核心架构设计

### 分层架构
```
┌─────────────────┐
│   用户界面层     │ ← CLI工具、配置向导、快速开始脚本
├─────────────────┤
│   业务逻辑层     │ ← 共享模块（PipelineManager、RetryManager等）
├─────────────────┤
│   数据访问层     │ ← 配置文件、元数据存储、输出文件
├─────────────────┤
│   技能执行层     │ ← 6个核心技能包 + 外部技能扩展
└─────────────────┘
```

### 模块化设计
- **高内聚**：每个模块负责单一职责
- **低耦合**：模块间通过清晰接口交互  
- **可扩展**：支持新技能包和功能扩展

## 🔧 核心功能模块

### 1. 共享核心模块 (`shared/`)

#### PipelineManager (296行)
- **流水线状态管理**：创建、更新、查询流水线
- **元数据跟踪**：YAML格式的完整执行历史
- **阶段状态监控**：pending/in_progress/completed/failed
- **错误统计分析**：记录错误类型、频率、解决方案

#### RetryManager (334行)
- **智能错误分类**：
  - 可重试错误：网络超时、API限制、内容审查失败
  - 不可重试错误：认证失败、IP白名单错误、无效配置
- **指数退避算法**：5s, 15s, 45s 重试间隔
- **重试建议生成**：针对不同错误类型的优化建议

#### UserConfirm (462行)
- **用户确认节点管理**：
  - SEO标题选择确认
  - 预览效果确认  
  - 最终发布确认
  - 修改/重试决策
- **交互式提示**：清晰问题描述、有限选项选择
- **确认历史记录**：用户选择跟踪和分析

#### ConfigLoader (345行)
- **多级配置优先级**：
  1. 环境变量
  2. .env文件
  3. 本地配置文件 (YAML > JSON)
  4. 全局配置文件 (~/.claude/env.json)
  5. 默认值
- **类型转换支持**：布尔值、整数、列表、字典
- **配置验证**：必填项检查、格式验证
- **模板生成**：自动生成配置模板

### 2. 技能包系统 (`skills/`)

#### 核心技能包 (6个)

**1. article-generator** - 文章生成器
- **功能**：生成技术博客文章，Markdown/Obsidian格式
- **特色**：
  - AI图片生成（Gemini API）
  - PicGo图床上传
  - YAML frontmatter + Obsidian callout
  - 代码示例 + 反AI痕迹写作风格
- **触发词**：`写文章`、`写一篇`、`article`

**2. content-reviewer** - 内容审查器
- **功能**：7维评分审查文章质量
- **评分维度**：
  - 可读性
  - 逻辑流
  - 标题吸引力
  - 事实准确性
  - AI痕迹检测
  - 平台适配性
  - 原创性
- **模式**：
  - 快速模式：低token消耗
  - 完整模式：深度审查，带截图证据
- **通过标准**：≥55分（总分70）
- **触发词**：`审查文章`、`内容审查`、`review article`

**3. wechat-article-converter** - 微信格式转换器
- **功能**：Markdown → 微信公众号HTML
- **主题支持**：
  - Python引擎：7个主题（Coffee/Tech/Warm等）
  - Go后端：9个主题（AI/API主题等）
- **高级功能**：
  - 草稿箱自动上传
  - 本地预览服务器
  - 图片压缩和优化
  - AI写作助手
- **触发词**：`转微信格式`、`微信排版`、`上传草稿箱`

**4. wechat-seo-optimizer** - SEO优化器
- **功能**：优化文章标题、摘要和关键词
- **核心能力**：
  - 标题A/B测试（5组方案）
  - 关键词密度分析
  - 摘要优化建议
  - 封面图推荐
- **标题公式库**：数字锚点、悬念缺口、痛点直击等8种模式
- **触发词**：`标题优化`、`SEO`、`取标题`

**5. content-planner** - 内容规划器
- **功能**：选题规划和内容日历
- **输出**：
  - 选题卡片（4-8个选题）
  - 月度排期表
  - 发布时间建议
- **触发词**：`选题`、`内容规划`、`内容日历`

**6. content-remixer** - 爆款拆解器
- **功能**：拆解爆款文章，生成创意积木
- **输出**：
  - 结构模式分析
  - 叙事手法提炼
  - 节奏模板提取
  - 可复用写作模板
- **触发词**：`拆解爆款`、`创意积木`、`remix文章`

#### 外部技能包 (需单独安装)

**content-repurposer** - 多平台分发器
- **功能**：一鱼多吃，文章→多平台适配
- **支持平台**：小红书、Twitter、知乎、Newsletter等
- **输出**：各平台专属格式

**content-analytics** - 数据分析器
- **功能**：公众号数据分析复盘
- **分析维度**：
  - 阅读量、点赞、分享数据
  - 增长趋势分析
  - 内容健康度诊断
- **数据来源**：微信后台导出数据 + 流水线元数据

### 3. 工具集 (`scripts/`)

#### pipeline_cli.py (403行) - 统一CLI工具
- **流水线管理**：创建、运行、监控、清理
- **状态查看**：实时状态监控、详细执行统计
- **数据导出**：JSON/YAML/CSV格式导出
- **测试功能**：组件测试、集成测试

#### config_wizard.py (403行) - 交互式配置向导
- **逐步引导配置**：Gemini API、微信公众号API、作者信息等
- **输入验证**：实时验证配置格式和有效性
- **环境检测**：自动检测Python版本、依赖安装情况
- **配置文件生成**：生成完整配置文件

#### quick_start.sh (200+行) - 快速开始脚本
- **一键安装**：检查环境、安装依赖、配置向导
- **环境检查**：Python 3.8+、pip、必要依赖
- **自动测试**：安装完成后自动测试所有组件
- **权限设置**：自动设置脚本执行权限

#### setup_permissions.py - 权限设置工具
- **跨平台兼容**：确保脚本可执行权限
- **权限修复**：从Git克隆后自动修复权限

### 4. Agent系统 (`agents/`)

#### content-pipeline - 内容流水线编排器
- **端到端协调**：协调全部技能完成从选题到发布的完整流程
- **8阶段流水线**：
  1. 选题规划 → 2. 文章生成 → 3. 内容审查 → 4. SEO优化
  5. 格式转换 → 6. 正式发布 → 7. 多平台分发 → 8. 数据复盘
- **智能调度**：根据用户需求自动选择执行路径

## 🚀 核心技术特性

### 1. 智能错误处理系统
- **分层错误分类**：致命错误、可恢复错误、用户决策错误
- **智能重试机制**：
  - 内容审查失败：最多重试3次
  - 网络/API错误：最多重试2次（指数退避）
  - 图片生成失败：可选跳过，手动添加图片
- **错误恢复策略**：从失败点继续执行，避免重复工作

### 2. 用户交互优化
- **关键确认节点**：
  - SEO标题选择（5个优化方案中选择）
  - 预览效果确认（查看微信格式预览）
  - 最终发布确认（发布前最后确认）
- **交互式提示**：
  - 清晰的问题描述
  - 有限的选项选择
  - 默认值和建议
  - 输入验证和错误提示

### 3. 数据持久化与监控
- **流水线元数据系统**：
  - 自动生成 `config/pipeline_metadata/{pipeline_id}.yaml`
  - 完整的执行状态跟踪
  - 各阶段耗时统计
- **执行统计**：
  - 错误次数和类型统计
  - 重试次数和成功率
  - 资源使用情况
- **调试信息**：
  - 详细的执行日志
  - 错误堆栈信息
  - 环境变量和配置状态

### 4. 性能优化
- **并行执行**：支持独立阶段的并行执行
- **缓存机制**：缓存配置和中间结果，减少重复计算
- **资源管理**：自动清理旧的流水线数据，可配置保留策略
- **增量更新**：支持从失败点继续执行，避免重复工作
